How "always latest" works:
  model.tag = "{{ .Values.model.tag }}" and modctl.alwaysPull = {{ .Values.modctl.alwaysPull }}.
  With tag=latest + alwaysPull=true, the init container re-pulls the newest weights
  into the cached volume on every pod start, so each run uses the latest model.

Reach the ZMQ inference server inside the cluster (GR00T PolicyClient):
  host = {{ include "gr00t-inference.fullname" . }}.{{ .Release.Namespace }}.svc
  port = {{ .Values.service.port }}

  from gr00t.policy.server_client import PolicyClient
  policy = PolicyClient(host="{{ include "gr00t-inference.fullname" . }}.{{ .Release.Namespace }}.svc", port={{ .Values.service.port }})
  action, info = policy.get_action(observation)

  kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "gr00t-inference.controlUi.fullname" . }} {{ .Values.controlUi.service.port }}:{{ .Values.controlUi.service.port }}
  then open http://localhost:{{ .Values.controlUi.service.port }}
  Stop = scale the inference deployment to 0 (GPU freed; weights stay cached).

  the inference pod's CPU/memory over time (metrics-server), and GPU allocation.
